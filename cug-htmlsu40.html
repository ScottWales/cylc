10.2 Built-in Job Submission Methods

Cylc has a number of built in job submission methods. If these do not suite your needs you can extend cylc with your own user-defined methods (below, Section 10.4).

  • background - run tasks directly in a background shell.
  • at - submit tasks to the rudimentary Unix at scheduler (atd must be running).

Tasks in a real forecasting system would normally be submitted to an industrial strength batch queue scheduler. Current built-in methods are:

  • loadleveler - submits task job scripts to loadleveler using the llsubmit command. Loadleveler directives provided in the suite.rc file:
    # SUITE.RC 
    [runtime] 
        [[__NAME__]] 
            [[[directives]]] 
                foo = bar 
                baz = waz

    are written to the top of the task job script like this:

    #!/bin/bash 
    # DIRECTIVES 
    # @ foo = bar 
    # @ baz = waz 
    # @ queue
  • pbs - submits task job scripts to PBS (or Torque) using the qsub command. PBS directives provided in the suite.rc file:
    # SUITE.RC 
    [runtime] 
        [[__NAME__]] 
            [[[directives]]] 
                -q = foo 
                -l = 'nodes=1,walltime=00:01:00'

    are written to the top of the task job script like this:

    #!/bin/bash 
    # DIRECTIVES 
    #PBS -q foo 
    #PBS -l nodes=1,walltime=00:01:00
  • sge - submits task job scripts to Sun Grid Engine using the qsub command. SGE directives provided in the suite.rc file:
    # SUITE.RC 
    [runtime] 
        [[__NAME__]] 
            [[[directives]]] 
                -cwd = ' ' 
                -q = foo 
                -l = 'h_data=1024M,h_rt=24:00:00'

    are written to the top of the task job script like this:

    #!/bin/bash 
    # DIRECTIVES 
    #$ -cwd 
    #$ -q foo 
    #$ -l h_data=1024M,h_rt=24:00:00

Note that multiple entries for the same PBS or SGE option must be comma-separated on the same line, in the suite.rc file. This is because repeating the option on another line will override the previous entry, not add to it. And, the right-hand side must be quoted to hide the comma from the suite.rc parser (commas are allowed only for list values; directives are treated as singular).

Note also that to get a naked option flag such as -cwd in SGE you must give a quoted blank space as the directive value.

Finally, default directives are provided to set the job name and stdout and stderr file paths. These can be overridden if you wish (but stdout/stderr directory paths should be set via the appropriate suite.rc item rather than using the directives here).
